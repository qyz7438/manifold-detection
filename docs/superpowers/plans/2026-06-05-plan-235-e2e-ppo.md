# Plan 2.35: 端到端 RLVR（PPO）

> **目标:** 2.33（RPN 采样）+ 2.34（bbox 探索）联合，PPO 稳定训练。RPN 和 box_head 同时 RL 更新。

> **前提:** 2.33 和 2.34 均成功。

---

## 设计

```
单张图 forward:
  1. backbone → FPN features（冻结）
  2. RPN 采样 proposal 集合 P（2.33 策略）
  3. 每个 proposal p ∈ P:
       box_head → Gaussian policy → 采样 delta → 候选框 b'
       候选框 b' → ROI FFT → reward(b')
  4. reward 组件:
       r = 0.3×HF_energy + 0.4×(1-entropy) + 0.3×phase_coherence
  5. PPO update:
       ratio = π_new(a|s) / π_old(a|s)
       clipped = clip(ratio, 0.8, 1.2) × advantage
       loss = -min(ratio×adv, clipped) + value_loss

  advantage = r - V(s)（value_head 做 baseline）
```

**冻结:** backbone。RPN + box_head + box_predictor + value_head 全训练。

**超参:** clip_ϵ=0.2, γ=0.99, value_loss_coef=0.5, K=20, M=10

**实现要点:**
- PPO buffer：存储 (state, action, reward, old_log_prob, advantage)
- value_head = small MLP on FPN pooled features
- 多 epoch PPO update（3-5 pass over buffer）
- 需要 `torch.no_grad()` 跑旧 policy 收集 buffer

---

## 验证

从 2.33+2.34 最佳 checkpoint 出发，10 epoch PPO × 3 seed = 3 组。
期望 AP75 超过单独 2.33 或 2.34——端到端 RL 覆盖完整检测链路。
